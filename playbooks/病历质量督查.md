# Playbook：口腔门诊病历质量督查

## 目的

完成一次对 `shhy.linkedcare.cn` 上所有可见患者电子病历的自动化质量督查，输出一份可在浏览器中查看的详细评分报告（HTML），报告中包含各患者得分、扣分明细、统计图表及需人工复核项列表。

## 前提条件

- Python 3.9+ 已安装
- 网络可访问 `https://shhy.linkedcare.cn`
- 项目目录可写，脚本会按需创建 `.tmp/`
- 环境变量 `MEDREC_USERNAME` 与 `MEDREC_PASSWORD` 已配置正确的登录账号和密码
- 执行前已运行 `bash scripts/setup_env.sh` 安装依赖

## 步骤

1. 【脚本调用】`bash scripts/setup_env.sh` — 检查并安装 Python 依赖与 Playwright Chromium 浏览器
2. 【脚本调用】`python scripts/crawl_records.py` — 自动登录并批量抓取所有患者病历，输出 `.tmp/records.json`
3. 【判断】`crawl_records.py` 的退出码：
   - 退出码 0 → 抓取成功且有患者，继续步骤 4
   - 退出码 2 → 抓取成功但工作台无患者，流程正常终止
   - 退出码 1 → 执行出错，进入步骤 3b 错误处理
4. 【脚本调用】`python scripts/score_records.py` — 读取病历 JSON，按督查表规则逐项评分，输出 `.tmp/scored_records.json`
5. 【脚本调用】`python scripts/generate_report.py` — 读取评分明细 JSON，生成带 ECharts 的 HTML 报告并自动打开浏览器

### 错误处理（步骤 3b）

- 若 `crawl_records.py` 因登录失败退出：检查 `src/config.py` 账号密码是否正确，网络是否通畅，重试步骤 2
- 若因页面加载超时退出：检查目标网站是否可访问，必要时增加 `config.py` 中的等待时间，重试步骤 2
- 连续失败 3 次 → 流程终止，报告错误日志路径

## 判断标准

- 某患者点击后无病历内容（页面未跳转或抓不到病历文本）：标记 `skipped=True`，`skip_reason="未提取到病历内容"`，最终报告显示"无病历"等级（评 0 分）
- 病历日期非今日：标记 `skipped=True`，`skip_reason="病历日期非今日: <date>"`，最终报告显示"无病历"等级（评 0 分）
- 病历日期识别失败（非 `yyyy-mm-dd hh:mm` 严格格式）：标记 `skipped=True`，`skip_reason="未识别到病历日期"`，最终报告显示"无病历"等级（评 0 分）。宁可漏评真实当日病历，也不让历史病历因日期识别异常被当成今日。
- 抓取过程中网络中断：当前版本不支持增量续跑，重新运行会全量重跑
- 评分时必填字段缺失：该字段对应检查项记 0 分，并在扣分原因中说明内容为空
- 现病史、既往史、诊断可缺失，缺失不扣分；若有内容则必须符合对应规则
- 辅助检查可缺失，如有则须含「日期 + 影像名 + 描述」三要素；缺失不扣分
- 辅助检查不为空时，相关影像必须有实际影像图片；辅助检查为空时，相关影像亦应为空
- 7-5（医嘱/告知）从医嘱字段中判断，不再从处置字段判断；当前规则集不单独评分 7-4
- 报告中"无病历"患者单独分组列出，展示患者姓名、ID 与跳过原因，便于人工核对
- 本流程线性执行，主要分叉为抓取失败后的重试策略

## 验证

- 报告 HTML 可在浏览器正常打开，无脚本错误
- 报告中的总患者数与 `records.json` 中的记录数一致
- 随机抽查 1-2 份病历的得分计算，与手工按督查表核对一致
- ECharts 图表（柱状图、饼图）正常渲染，数据与表格一致
