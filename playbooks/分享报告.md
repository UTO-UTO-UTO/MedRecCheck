# Playbook：分享病历评分报告

## 目的
通过链接让他人能够在线查看报告、人工复核评分、下载 PNG 截图。

## 前提条件

### GitHub Pages 方案（长期稳定推荐）

- GitHub 仓库已创建，项目代码已推送
- 仓库 Settings → Pages → Source → **Deploy from a branch** → 选择 `gh-pages` 分支，目录 `/ (root)`
- 仓库 Settings → Secrets and variables → Actions → 添加以下 Secrets：
  - `MEDREC_USERNAME`：shhy.linkedcare.cn 登录用户名
  - `MEDREC_PASSWORD`：shhy.linkedcare.cn 登录密码
  - （可选）`LLM_API_KEY`：LLM API Key，不配则使用规则评分

### 本地临时分享方案

- 报告文件已生成（`.tmp/report.html` 存在）
- Python 3 已安装

---

## 步骤

### 方式一：GitHub Actions 全自动（推荐）

**首次配置**：

1. 确认项目已推送到 GitHub 仓库
2. 在仓库 Settings → Pages 中配置 source 为 `gh-pages` 分支
3. 在仓库 Settings → Secrets and variables → Actions → Repository secrets 中添加：
   - `MEDREC_USERNAME`
   - `MEDREC_PASSWORD`
4. 首次登录可能需要在本地执行一次（见下方说明）

**定时自动运行**：

- 每天北京时间 08:30 自动执行（由 `.github/workflows/deploy.yml` 中的 cron 控制）
- 部署完成后，报告自动更新到固定链接：`https://你的用户名.github.io/仓库名`

**手动触发**：

1. 打开仓库页面 → Actions → 病历评分 & 部署 → Run workflow
2. 等待执行完成后，访问固定链接即可

**验证**：

- 打开仓库 Actions 页面，确认 workflow 执行成功
- 访问 `https://你的用户名.github.io/仓库名`，确认报告正常加载

### 方式二：本地部署到 GitHub Pages（保留灵活控制）

```bash
# 1. 执行评分流程
bash scripts/run_full_pipeline.sh

# 2. 推送到 GitHub Pages
scripts/deploy_to_github.sh
```

或一键完成：

```bash
DEPLOY_TO_GITHUB=1 bash scripts/run_full_pipeline.sh
```

### 方式三：本地服务器 + 临时公网链接

```bash
scripts/serve_report.sh --ngrok
```

## 判断标准

### GitHub Actions 方案

- 若 CI 首次运行登录失败（需要验证码等交互）：先在本地执行 `python scripts/crawl_records.py` 完成登录并保存 `.tmp/state.json`，将该文件提交到仓库或通过 GitHub Secrets 传递
- 若 `crawl_records.py` 退出码为 2（无患者）：视为正常，不继续评分和部署
- 若评分流程中任一环节失败：查看 Actions 日志定位原因

### 本地方案

- 本流程线性执行，无需预定义判断标准

## 验证

- [ ] 固定链接可正常访问，页面完整渲染
- [ ] 每条评分后显示「+分」「-分」按钮，点击后分数和等级实时更新
- [ ] 点击「下载 PNG」能正常保存截图
- [ ] GitHub Actions workflow 定时执行无异常
- [ ] 重新部署后，链接内容已更新

## 修改定时

编辑 `.github/workflows/deploy.yml` 中的 cron 表达式：

```yaml
schedule:
  - cron: '30 0 * * *'   # UTC 00:30 = 北京时间 08:30
```

| cron 表达式 | 含义 |
|-------------|------|
| `30 0 * * *` | 每天北京时间 08:30 |
| `0 1 * * *` | 每天北京时间 09:00 |
| `30 22 * * *` | 每天北京时间 06:30（次日） |
