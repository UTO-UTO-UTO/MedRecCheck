# Playbook：分享病历评分报告

## 目的
通过链接让他人能够在线查看报告、人工复核评分、下载 PNG 截图。

## 前提条件
- 报告文件已生成（`.tmp/report.html` 存在）
- Python 3 已安装
- 项目已关联 GitHub 仓库（`git remote add origin ...`）
- GitHub 仓库已开启 Pages（首次部署需手动配置）

## 步骤

### 方式一：仅本地访问（自己查看）

```bash
scripts/serve_report.sh
```

自动打开浏览器访问 `http://localhost:8080`。

### 方式二：生成公网链接（临时分享）

```bash
scripts/serve_report.sh --ngrok
```

执行后会输出两个地址：
- **本地地址**：`http://localhost:8080`
- **公网链接**：`https://xxxx.ngrok-free.app`（示例）

将公网链接通过微信/钉钉/邮件发给他人即可。

### 方式三：部署到 GitHub Pages（推荐长期使用）

#### 首次配置（只需做一次）

1. **确认仓库已推送到 GitHub**

   ```bash
   git remote -v
   # 应显示类似：
   # origin  https://github.com/你的用户名/仓库名.git (fetch)
   ```

   如未配置，请先推送：
   ```bash
   git remote add origin https://github.com/你的用户名/仓库名.git
   git push -u origin main
   ```

2. **在 GitHub 开启 Pages**

   打开仓库页面 → Settings → Pages → Source：
   - 选择「Deploy from a branch」
   - 分支选择 `gh-pages`
   - 点击 Save

   > 首次可能需要先创建 `gh-pages` 分支。可先运行一次部署脚本，或手动创建空分支：
   > ```bash
   > git checkout --orphan gh-pages
   > git rm -rf .
   > touch README.md
   > git add README.md
   > git commit -m "init"
   > git push origin gh-pages
   > git checkout main
   > ```

3. **检查访问权限**

   - 公开仓库：任何人都能访问
   - 私有仓库：需要升级到 GitHub Pro 或使用其他方案

#### 日常部署（评分完成后执行）

```bash
# 方式 A：手动部署（评分完成后单独执行）
scripts/deploy_to_github.sh

# 方式 B：一键执行全流程（评分 + 部署）
bash scripts/run_full_pipeline.sh  # 评分流程
scripts/deploy_to_github.sh        # 推送到 GitHub Pages
```

部署成功后会输出固定链接：
```
访问链接: https://你的用户名.github.io/仓库名
```

这个链接**每天都是一样的**，你只需发给相关人员一次，之后每次部署内容会自动更新。

#### 高级选项

```bash
# 指定报告文件
scripts/deploy_to_github.sh --report .tmp/report_2025-01-15.html

# 自定义提交信息
scripts/deploy_to_github.sh --message "今日复查报告"

# 预览模式（不实际推送）
scripts/deploy_to_github.sh --dry-run
```

### 方式四：指定端口或报告文件（本地）

```bash
# 使用自定义端口
scripts/serve_report.sh --port 3000 --ngrok

# 指定其他报告文件
scripts/serve_report.sh --report .tmp/report_2025-01-15.html
```

## 判断标准
- 本流程线性执行，无需预定义判断标准。
- 若报告文件不存在，本地 serve 脚本会自动尝试重新生成。
- 若 ngrok 未安装，脚本会尝试通过 Homebrew 安装（macOS）。
- 若 GitHub Pages 未配置，部署脚本会提示设置步骤。
- 若 GitHub Pages 首次访问报 404，等待 1-2 分钟后重试（GitHub 需要时间来部署）。

## 验证
- [ ] 本地浏览器能正常打开报告页面
- [ ] 页面中每条评分后显示「+分」「-分」按钮
- [ ] 点击按钮后分数和等级实时更新
- [ ] 点击「下载 PNG」能正常保存截图
- [ ] （ngrok）手机/其他网络能打开公网链接
- [ ] （GitHub Pages）固定链接可正常访问，刷新后仍显示
- [ ] （GitHub Pages）重新部署后，固定链接内容已更新
