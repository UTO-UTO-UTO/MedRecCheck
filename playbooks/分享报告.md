# Playbook：分享病历评分报告

## 目的
通过链接让他人能够在线查看报告、人工复核评分、下载 PNG 截图。

## 前提条件

- 报告文件已生成（`.tmp/report.html` 存在）
- Python 3 已安装

---

## 步骤

### 方式一：本地定时执行 + GitHub Pages（最可靠）

如果你有 Mac 或 Linux 服务器可以定时运行评分流程，这是最稳定的方案。

**1. 安装定时任务（macOS）**

```bash
# 每天 22:00 自动执行：抓取 → 评分 → 部署
bash scripts/install_launchd.sh

# 自定义时间（如每天 08:30）
bash scripts/install_launchd.sh 8 30
```

**2. 检查状态**

```bash
launchctl list | grep medrec          # 查看任务是否运行
tail -f .tmp/launchd.log              # 查看执行日志
launchctl start com.medrec.report     # 手动触发一次
```

**3. 停止或卸载**

```bash
launchctl unload ~/Library/LaunchAgents/com.medrec.report.plist   # 停止
rm ~/Library/LaunchAgents/com.medrec.report.plist                 # 卸载
```

安装后，任务会每天自动执行。完成后报告自动推送到 GitHub Pages 固定链接。

**4. 首次手动部署（确认 GitHub Pages 配置）**

```bash
# 在 GitHub 仓库设置中配置 Pages（只需一次）
# Settings → Pages → Source → Deploy from a branch → gh-pages → / (root)

# 执行一次全流程
bash scripts/run_full_pipeline.sh
scripts/deploy_to_github.sh
```

### 方式二：GitHub Actions 定时（备选）

> **已知限制**：目标网站有反爬机制，GitHub Actions 的无头浏览器可能无法登录。此时 workflow 会回退到历史数据，不会中断部署。

**配置**：

1. 推送代码到 GitHub
2. 仓库 Settings → Pages → Source → `gh-pages` 分支
3. 仓库 Settings → Secrets and variables → Actions → 添加 `MEDREC_USERNAME` 和 `MEDREC_PASSWORD`
4. 仓库 Actions 页面手动触发一次验证

### 方式三：本地服务器 + 临时链接

```bash
scripts/serve_report.sh --ngrok
```

---

## 链接格式

| 方案 | 链接 |
|------|------|
| GitHub Pages | `https://你的用户名.github.io/仓库名` |
| ngrok | `https://xxxx.ngrok-free.app`（每次重启变化）|
| 本地 | `http://localhost:8080` |

将链接发给相关人员，对方打开后即可：
- 查看全部患者评分详情
- 点击 +分 / -分 进行人工复核
- 点击「下载 PNG」保存报告截图

---

## 判断标准

- **CI 抓取失败**（需验证码）→ workflow 自动回退到历史数据，不中断部署
- **CI 连续失败超过一周**→ 切换到方式一（本地定时执行）
- 本流程线性执行，无需预定义判断标准

## 验证

- [ ] 固定链接可正常访问
- [ ] 每条评分后显示「+分」「-分」按钮，点击后分数和等级实时更新
- [ ] 点击「下载 PNG」能正常保存截图
- [ ] （本地定时）日志 `.tmp/launchd.log` 无异常
- [ ] 重新部署后，刷新链接内容已更新
